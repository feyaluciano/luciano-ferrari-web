La Historia Detrás de Esta Web

Aquí te contaré cómo construí este sitio web personal y portafolio profesional, desde la idea inicial hasta el despliegue final.

El Por Qué

La idea de desarrollar esta web nació de una doble curiosidad y una necesidad práctica. Por un lado, me impulsó un profundo deseo de **explorar los límites de la colaboración con la Inteligencia Artificial** en el desarrollo de software. Quería poner a prueba la capacidad de la IA para construir una aplicación web moderna, utilizando algunas de las últimas tecnologías del mercado como Angular 19 y Tailwind CSS, **escribiendo el menor código posible de mi parte**. Mi enfoque fue guiar a la IA a través de prompts detallados, pasarle imágenes de un template estático y solicitarle que realizara la maquetación en HTML y Tailwind. Para esta experimentación, evalué y comparé el rendimiento de diferentes modelos de IA, incluyendo **OpenAI, Gemini y DeepSeek**, analizando cuál resolvía mejor las tareas de desarrollo planteadas.

Por otro lado, esta web sirve como una **ventana profesional hacia reclutadores y colegas**. Es un espacio donde no solo muestro mis proyectos, sino que también evidencio mi actividad y conocimiento en ciertas tecnologías, la aplicación de patrones de diseño, mi compromiso con el código limpio y mi interés en la mejora continua.

Finalmente, esta plataforma se concibió como un **ejercicio de aprendizaje continuo y una bitácora personal**. Mi objetivo es tener un lugar centralizado donde pueda documentar y organizar los conocimientos, enlaces y recursos importantes que voy adquiriendo en mi día a día laboral, transformando mis aprendizajes en recursos valiosos para mí y, potencialmente, para la comunidad.

El Cómo: Tecnologías y Proceso

La Colaboración con la IA: Un Diálogo Inicial

El punto de partida de este proyecto fue un ejercicio de "prompt engineering". Mi primer paso fue enviar un prompt inicial que, honestamente, no estaba del todo bien redactado, solicitando a la IA que lo puliera y lo hiciera más claro y preciso. La IA demostró una gran capacidad para entender mi intención y refinar mi solicitud.

Este fue el prompt final, ya mejorado por la IA, que sirvió como hoja de ruta para todo el desarrollo:

"Hola, estoy listo para comenzar a desarrollar mi sitio web personal y portafolio profesional. Mi objetivo es crear una aplicación moderna con Angular (última versión) y Tailwind CSS. Este sitio web incluirá las siguientes secciones principales:

  • Currículum Vitae (CV): Presentación de mi experiencia profesional, habilidades y educación.
  • Proyectos: Una galería o listado detallado de los proyectos de desarrollo de software en los que he trabajado, con descripciones, tecnologías utilizadas y enlaces (si aplica).
  • Bitácora/Recursos: Una sección para compartir enlaces interesantes, artículos, o pequeñas notas sobre desarrollo de software y tecnologías que encuentro útiles.

Ya cuento con una plantilla HTML/CSS estática que he desarrollado previamente y que servirá como base visual para el diseño del sitio. Mis objetivos clave para este proyecto son:

  • Desarrollo y Despliegue: Crear la aplicación Angular, integrar la plantilla existente, y finalmente desplegarla en Vercel a partir de un repositorio en GitHub.
  • Aprendizaje Profundo de Angular y Buenas Prácticas: A medida que construimos el sitio, quiero que me guíes y me enseñes:Estructura del Proyecto: Explícame la lógica detrás de la estructura de directorios que propongas, por qué es escalable y mantenible.
  • Conceptos Modernos de Angular: Detalla el uso, importancia y ventajas de características como componentes standalone, signals, servicios, enrutamiento avanzado, lazy loading, manejo de formularios reactivos, e inyección de dependencias.
  • Patrones de Diseño: Identifica y explícame los patrones de diseño (ej. estructurales, creacionales, de comportamiento) que vayamos implementando, tanto a nivel de componente como de aplicación.
  • Integración con Tailwind CSS: Muéstrame las mejores prácticas para configurar y utilizar Tailwind CSS dentro de un proyecto Angular, incluyendo la personalización y optimización (PurgeCSS).
  • Calidad de Código y Pruebas: Si es posible, introduce conceptos básicos de testing en Angular.
  • Optimización para el Despliegue: Consejos para optimizar el build final para Vercel.

Para empezar, por favor, ayúdame con los siguientes pasos iniciales:

  • ¿Cómo creamos un nuevo proyecto Angular (última versión) configurado desde cero para usar componentes standalone por defecto?
  • ¿Cuál sería la estructura de directorios inicial recomendada para un proyecto de este tipo, pensando en las secciones que te mencioné (CV, Proyectos, Bitácora) y la escalabilidad?
  • ¿Cómo integramos Tailwind CSS de manera efectiva en este nuevo proyecto Angular?
  • Una vez configurado el proyecto base y Tailwind, ¿cuál sería el primer paso para empezar a migrar/integrar mi plantilla HTML/CSS existente? Podríamos empezar por el layout principal (cabecera, pie de página, contenedor principal).

Ya instalé Angular y creé el proyecto llamado luciano-ferrari-web"

A partir de este prompt detallado, la IA pudo guiarme a través de los pasos iniciales, incluyendo la creación del proyecto Angular con componentes standalone y la integración de Tailwind CSS.

Estructura del Proyecto: Modularidad y Escalabilidad

Un pedido clave a la IA fue su recomendación sobre la estructura de directorios. Me proporcionó una organización lógica con carpetas como core (para servicios esenciales), shared (para componentes reutilizables como la navegación o tarjetas) y features (para encapsular las secciones principales como CV, Proyectos y Bitácora). Esta estructura fue elegida y explicada por la IA por su contribución a la modularidad, la facilidad de mantenimiento y la escalabilidad, facilitando el lazy loading de módulos y el trabajo en equipo.

Tecnologías Clave del Proyecto

Además de la IA como co-piloto, este proyecto se construyó sobre pilares tecnológicos sólidos:

  • **Angular 19 (Pre-lanzamiento) y TypeScript**: Elegí la última versión de Angular para explorar sus nuevas funcionalidades y la API de componentes standalone, lo que me permitió escribir código más modular y optimizado. TypeScript garantiza la robustez y la legibilidad del código.
  • **Tailwind CSS**: Para el estilado, opté por Tailwind CSS. Su enfoque de "utility-first" me permitió construir interfaces responsivas de manera extremadamente rápida y mantener una coherencia visual sin escribir CSS personalizado extenso.
  • **Vercel y GitHub**: El despliegue se gestiona a través de Vercel, conectado directamente a un repositorio en GitHub. Esto facilita un flujo de trabajo de integración y despliegue continuo (CI/CD), donde cada push al repositorio dispara un nuevo despliegue en producción.

Patrones de Diseño y Código Limpio

A lo largo del desarrollo, apliqué activamente varios patrones de diseño y principios de código limpio. Por ejemplo, utilicé la **Inyección de Dependencias** de Angular para gestionar los servicios (como ProjectsService y NotebookSheetService), lo que facilita la prueba y el mantenimiento del código. La estructura de carpetas core, shared y features es un reflejo del principio de **Separación de Intereses** y modularidad.

Aprendizajes y Reflexiones

La experiencia de desarrollar esta web me ha dejado valiosos aprendizajes sobre la colaboración con la Inteligencia Artificial. La IA ha demostrado ser una herramienta de apoyo inmensamente útil, capaz de acelerar el proceso de desarrollo de manera significativa. Sin embargo, un hallazgo interesante fue cómo **algunos modelos de IA resolvieron mejor ciertas tareas que otros**, evidenciando las fortalezas y debilidades particulares de cada uno en diferentes contextos.

Es innegable que para aprovechar al máximo estas herramientas, no solo es crucial **saber CÓMO pedirle a la IA que desarrolle lo solicitado**, formulando prompts claros y precisos, sino también, y esto a menudo pasa desapercibido, **saber QUÉ pedirle**. La capacidad de discernir qué partes del desarrollo son delegables a la IA y cuáles requieren un enfoque humano, junto con la habilidad para estructurar esas peticiones, se convierte en una habilidad fundamental para el desarrollador moderno.

Este desarrollo se realizó principalmente a través de la interacción directa con prompts en modelos de lenguaje. No obstante, anticipo un futuro cercano donde la integración de estas capacidades directamente en los entornos de desarrollo será estándar. De hecho, tengo planes de empezar a explorar desarrollos similares utilizando entornos como **Windsurf** o **Cursor**, que ya incorporan chats y asistentes de IA directamente en el IDE. Lamento que Visual Studio, mi IDE preferido para el desarrollo de Backend con .NET, aún no cuente con este tipo de chats incorporados, lo que sin duda aumentaría la eficiencia. Pero, ¿quién dice que pronto no tendremos algo así? La evolución es constante.

Finalmente, en cuanto a la adopción de modelos de IA de pago, aún no estoy seguro. Lo que he necesitado resolver hasta ahora lo he logrado con los modelos gratuitos, lo cual es una gran ventaja. Sin embargo, el panorama de la IA avanza rápidamente y las capacidades de los modelos de pago son cada vez más atractivas. Quién sabe si algún día, o cuándo, la necesidad de mayor potencia o funcionalidades específicas me lleve a considerar una suscripción. Por ahora, sigo explorando.
